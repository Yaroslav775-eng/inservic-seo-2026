# QA-loop для ежедневного усиления страниц SK SHINE

Дата: 2026-07-23

## Зачем нужен этот этап

Первая версия прототипа показывает направление. Второй проход делает страницу сильной: мы намеренно критикуем черновик, ищем слабые места и только потом отправляем собственнику согласовательную версию.

## Порядок по каждой странице

1. Собрать текущую страницу и SERP/конкурентов.
2. Подготовить первую мягкую текстовую правку без изменения визуального шаблона.
3. Провести SEO-ревью черновика.
4. Провести AI-ревью черновика.
5. Провести конкурентное ревью.
6. Усилить страницу по найденным замечаниям.
7. Обновить прототип с аккуратной подсветкой изменений.
8. Обновить `QA Loop` и `QA Notes` в очереди.
9. Запушить финальную согласовательную версию на GitHub Pages.

## SEO-ревью

- H1 содержит услугу и город без переспама.
- Title и description соответствуют интенту и могут улучшить CTR.
- Первый экран за 5 секунд объясняет услугу, пользу, город и следующий шаг.
- Есть естественные вхождения основного и вторичных запросов.
- Есть блоки `що входить`, `що не входить`, `ціна/фактори`, `кому підходить`, `FAQ`.
- Нижний текст не выглядит как SEO-полотно и помогает принять решение.

## AI-ревью

- Страница дает короткие ответы на вопросы, которые AI может цитировать.
- В тексте четко отделены базовая услуга, допработы и ограничения.
- Цена объяснена через факторы, а не скрыта в общих словах.
- Есть локальность: Хмельницький, выезд, объект/адрес, если уместно.
- FAQ видим на странице и может быть поддержан schema.

## Конкурентное ревью

- Проверена локальная выдача по основному запросу.
- Выписаны 2-3 сильных конкурента/агрегатора.
- Понятно, где конкуренты сильнее: цена, отзывы, гарантии, структура, FAQ, доверие.
- В прототип добавлены только те усиления, которые не ломают текущий сайт.

## Коммерческий и UX-ревью

- Не обещаем невозможного и не пишем гарантии, которых бизнес не подтвердил.
- Подсветка изменений мягкая и выглядит как согласование текста, а не техническая ошибка.
- CTA говорит, что делать дальше.
- Правки можно перенести в WordPress без редизайна.

## Критерий готовности

Страница получает `Approval Ready`, только если:

- QA-loop пройден;
- замечания после критики внесены;
- есть публичный прототип с подсветкой;
- есть WP-ready текст;
- есть checklist/schema notes;
- очередь и трекер обновлены.
